## Introdução

A verificação é uma fase crucial no desenvolvimento de um projeto. Durante essa etapa, os artefatos gerados são analisados para garantir que atendem aos requisitos estabelecidos. O documento a seguir tem como objetivo apresentar de forma concisa e clara a metodologia adotada nas verificações realizadas tanto para o [grupo 4 - Cinemark](https://github.com/Interacao-Humano-Computador/2024.2-Cinemark) quanto para o [grupo 5 - CDMOJ](https://github.com/Interacao-Humano-Computador/2024.2-CDMOJ).

## Metodologia

A abordagem escolhida para a verificação é a inspeção. Criada por Fagan na IBM em 1976 para revisar códigos de software, essa técnica consiste em uma avaliação formal dos artefatos com o objetivo de identificar defeitos. A revisão é conduzida com base em uma lista de verificação, contendo os defeitos mais frequentes que devem ser localizados, analisados e classificados. É importante destacar que o autor principal do artefato não deve ser o responsável por sua leitura. As Tabelas 1 a XX resumem os responsáveis pela criação dos checklists para cada artefato, abrangendo os artefatos de planejamento, a execução das verificações e os resultados tanto para o próprio grupo quanto para o grupo 2.
